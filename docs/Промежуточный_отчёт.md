# Промежуточный отчёт о проделанной работе
### Проект «Scanner-SPBSTU» — автоматизированный комплаенс-скрининг

## 1. Задача

Создать систему, которая:
1. **автоматически собирает** единый перечень запрещённых/контролируемых
   организаций и лиц из официальных государственных реестров РФ;
2. **находит их упоминания** на сайтах СПбПУ (включая все поддомены);
3. **отслеживает обновления** источников, чтобы перечень был актуальным.

## 2. Что сделано

Реализованы три самостоятельные подсистемы поверх общей инфраструктуры
(единый HTTP-слой с обходом особенностей госсайтов, логирование, конфигурация).
Всё управляется YAML-конфигами — добавление источника, сайта или бренда не
требует правки кода.

### 2.1. Сбор перечня — `main.py`
Загрузка → очистка → нормализация → объединение → дедупликация → контроль
качества → выгрузка. Подключены **4 официальных источника**:

| Источник | Формат на сайте | Файл-загрузчик |
|---|---|---|
| Минюст — экстремистские организации | нумерованный HTML-список | `loaders/html_list_loader.py` |
| ФСБ — террористические организации | HTML-таблица | `loaders/web_loader.py` |
| Росфинмониторинг — террористы и экстремисты | список из 22 700+ элементов | `loaders/html_list_loader.py` |
| Минюст — реестр иностранных агентов | JSON-API (1 200+ записей) | `loaders/api_loader.py` |

Каждый источник имел свой «подводный камень» (SSL-сертификаты НУЦ РФ,
блокировки без браузерного заголовка, динамическая подгрузка иноагентов через
скрытый API, кавычки и скобки внутри названий) — всё решено и
задокументировано в [README.md](../README.md).

**Результат:** единая таблица из **~2 000 записей** (организации и иностранные
агенты) в трёх форматах — `output/registry_merged.{xlsx,csv,db}`. Обработка:
нормализация дат/ИНН, дедупликация без учёта регистра, объединение одной
сущности из нескольких реестров с перечнем всех её категорий.
**Физлица-террористы/экстремисты целенаправленно исключены** (на сайтах дают
лишь ложные срабатывания по однофамильцам); организации этих категорий и
физлица-иностранные агенты сохранены. Логика — в `transformers/` (очистка,
нормализация, объединение) и `exporters/` (xlsx/csv/sqlite).

### 2.2. Скринер сайтов — `scan.py` + `scanner/`
- **`scanner/discovery.py`** — обнаруживает все подразделения и поддомены из
  официальной структуры СПбПУ (**52 хоста**: институты, высшие школы,
  лаборатории, центры, кафедры, сервисы).
- **`scanner/sitemap.py`** — перечисляет все страницы каждого сайта через
  `sitemap.xml` — вежливо и полно (всего **~73 200 страниц**).
- **`scanner/crawler.py`** — обход с соблюдением `robots.txt`, паузами,
  поиском в т.ч. по ссылкам (ловит соцсети-иконки без подписи).
- **`scanner/matcher.py`** — поиск вхождений всех имён перечня за один проход
  по странице (алгоритм **Ахо-Корасик**), с проверкой границ слов, оценкой
  достоверности (организации/редкие имена — высокая; частые ФИО — «возможен
  тёзка») и **алиасами**: одна запрещённая сущность в разных написаниях на
  латинице и кириллице (Instagram/Инстаграм, Facebook/Фейсбук и т.п.).
- **`scanner/report.py`** — отчёт в CSV/XLSX с подсветкой достоверности и
  листом «Инфо» с правовой оговоркой.

**Подтверждено на живых данных:** на сайте Института промышленного менеджмента
(`ie.spbstu.ru`) найдены ссылки на **Facebook (Meta — запрещена в РФ)** в
сквозном футере; при полном прогоне выявлено **722 страницы** с упоминаниями
Meta на 13 поддоменах, а также упоминания иностранных агентов («Голос Америки»,
The Bell).

### 2.3. Мониторинг и расписание
- **`watch_sources.py`** — ежедневно проверяет источники по их «признакам
  актуальности» (ETag/Last-Modified у ФСБ, `lastModified` реестра у иноагентов,
  число записей + хеш у остальных) и сообщает, нужно ли пересобирать перечень.
- **`daily_watch.sh`** (ежедневно) и **`weekly_scan.sh`** (еженедельно, полный
  обход Политеха) + установщик расписания **`install_schedule.sh`**.

## 3. Надёжность и ответственность
- **Health-check** (`config.yaml`: `min_rows`, `min_total_rows`): если сайт
  сменит вёрстку и парсер вернёт мало/мусор — источник исключается, а при
  слишком малом итоге экспорт отменяется, прежний валидный перечень
  сохраняется.
- **Правовая оговорка** ([DISCLAIMER.md](../DISCLAIMER.md)): данные из публичных
  реестров «как есть», категории требуют заверения комплаенс-службой, совпадения
  по ФИО — кандидаты для ручной проверки, а не идентификация.
- Устойчивость к сбоям (ошибка одного источника/страницы не валит процесс),
  возобновляемость длительного обхода.

## 4. Статус и дальнейшие шаги
Рабочий, проверенный на живых источниках MVP; код — в репозитории
`Scanner-SPBSTU`. К полной сдаче планируется добавить: автоматические тесты +
CI, фиксацию версий зависимостей; при необходимости — реестр «нежелательных
организаций» и расширение списка брендов.
