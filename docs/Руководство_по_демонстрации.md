# Руководство по демонстрации (локальный запуск)

Памятка для показа заказчику. Весь показ — **~7–10 минут**, три шага.

---

## 0. Подготовка (один раз, заранее — НЕ на показе)

```bash
cd "/Users/artemy/Desktop/практ/registry_merger"
python3 -m venv .venv          # если ещё нет
source .venv/bin/activate
pip install -r requirements.txt
```

> Окружение уже создано на рабочей машине. Перед показом достаточно:
> ```bash
> cd "/Users/artemy/Desktop/практ/registry_merger" && source .venv/bin/activate
> ```

**Важно:** в каталоге `output/` уже лежат готовые файлы прошлых прогонов
(`registry_merged.xlsx`, `scan_demo.xlsx`). Если на показе будет плохой
интернет или госсайты «придержат» запросы — можно просто **открыть готовые
файлы**, не запуская заново. Живой запуск — по желанию.

---

## Шаг 1. Сборка единого перечня (~1 мин)

```bash
python main.py --config config.yaml
```
Что говорить: «Скрипт сам сходил в 4 официальных реестра (Минюст, ФСБ,
Росфинмониторинг), очистил, объединил и дедуплицировал данные».

Показать результат:
```bash
open output/registry_merged.xlsx        # ~2 000 строк: № | Наименование/ФИО | Тип
open output/DISCLAIMER.txt              # правовая оговорка + источники
```
Подчеркнуть: 3 колонки, объединение дублей, у одной сущности — все её категории.
В перечень входят **организации и иностранные агенты** (~2 000). Физлица-
террористы/экстремисты намеренно исключены — на сайтах они дают только ложные
срабатывания по однофамильцам.

> Если какой-то госсайт вернул ошибку — это нормально: скрипт пропустит
> источник и продолжит (устойчивость к сбоям), а health-check не даст
> перезаписать перечень неполными данными.

---

## Шаг 2. Скрининг сайта Политеха (~1–2 мин) — ГЛАВНОЕ

Быстрый демонстрационный прогон по одному институтскому сайту:
```bash
python scan.py --config scan_demo.yaml
```
Показать результат:
```bash
open output/scan_demo.xlsx
```
Что показать в файле:
- лист **matches** — найдены вхождения **Meta (Facebook/Instagram) — запрещена
  в РФ**, с подсветкой достоверности (зелёный = высокая), URL и контекстом.
  Это реальные ссылки на Facebook в футере страниц `ie.spbstu.ru`;
- лист **Инфо** — правовая оговорка.

Что говорить: «Система за один проход ищет все организации/бренды перечня (за счёт алгоритма Ахо-Корасик масштабируется до десятков тысяч), понимает разные
написания (Instagram/Инстаграм/ссылка на facebook.com), видит даже иконки
соцсетей без подписи, и сразу расставляет приоритеты по достоверности».

> Полный обход **всех** сайтов СПбПУ (≈73 000 страниц) — это многочасовая
> фоновая задача (`scan_config.yaml`), её на показе не запускаем. Можно
> упомянуть её результат: 722 страницы с Meta на 13 поддоменах.

## Шаг 3. Мониторинг актуальности (~30 сек)

```bash
python watch_sources.py --config config.yaml
```
Показать вывод: по каждому источнику — 🟢 «актуально» или 🔴 «ОБНОВИТЬ».

Что говорить: «Каждый день система сама проверяет, изменились ли реестры
(по дате/ETag/числу записей), и подсказывает, когда пересобирать перечень.
Проверка ставится в расписание одной командой `./install_schedule.sh`».

---

## Запасной план (если нет сети / госсайты недоступны)
1. Шаг 1 → вместо запуска `open output/registry_merged.xlsx` (готовый файл).
2. Шаг 2 → `open output/scan_demo.xlsx` (готовый отчёт с находками Meta).
3. Шаг 3 → показать `output/source_watch_report.json` (прошлый результат).

## Короткая шпаргалка команд
```bash
cd "/Users/artemy/Desktop/практ/registry_merger" && source .venv/bin/activate
python main.py --config config.yaml          # 1. собрать перечень
python scan.py --config scan_demo.yaml        # 2. скрининг (демо)
```
